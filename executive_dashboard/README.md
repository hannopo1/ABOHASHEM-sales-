# لوحة الأداء التنفيذي المالي — أبو هاشم للحوم (يونيو ٢٠٢٦)

لوحة تحكم تنفيذية تفاعلية بمظهر داكن premium، عربية بالكامل (RTL)، تعمل **بلا خادم وبلا
إنترنت** (كل المكتبات مُضمَّنة محليًا في `vendor/`). تركّز على مبيعات ومديونية وتحصيل شهر
**يونيو ٢٠٢٦**، وتُبنى بالكامل من الملفات المصدرية في هذا المستودع — لا رقم واحد مُختلَق.

## المخرجات (Deliverables)

| ملف | الوصف |
|---|---|
| `index.html` | اللوحة التفاعلية (٩ أقسام) — افتحها مباشرة في المتصفح |
| `style.css` | السمة الداكنة premium (glassmorphism) + استجابة + وضع طباعة |
| `script.js` | منطق العرض: ECharts/Plotly، DataTables، الفلترة المتقاطعة، التنقيب |
| `data.js` | البيانات المُجمّعة مسبقًا (`window.DASH`) — يُولّدها `build.py` |
| `processed_data.csv` | بنود فواتير يونيو بعد التنظيف |
| `insights.json` | التعليقات التنفيذية الآلية (ماذا/لماذا/مخاطر/فرص/إجراء/أولوية) |
| `executive_summary.pdf` | ملخص تنفيذي من صفحة واحدة (عربي مُشكّل) |

## التشغيل / إعادة التوليد

```bash
pip install -r executive_dashboard/requirements.txt
python3 executive_dashboard/build.py        # يطبع تقرير تحقّق بكل الفحوص PASS
# ثم افتح executive_dashboard/index.html في أي متصفح
```

`build.py` يقرأ فواتير يونيو المصدرية (`فواتير_المبيعات_يونيو_2026-1.md`) وملفات الأبعاد
المُعالجة (`data/processed/`, `analysis/data_2026_06/`)، ويُجري كل التجميع على **Polars** (يتوسّع
دون تغيير إلى ملايين الصفوف)، ثم يكتب كل المخرجات ويطبع تقرير التحقّق.

## سلّم الحافز (متغيّر واحد)

يُضبط بالكامل من `src/config.py`:

```python
BONUS_RULES = [(0.70,0.00),(0.80,0.01),(0.90,0.02),(0.95,0.03),(1.01,0.05)]
```

يُقرأ: تحصيل < ٧٠٪ ⇒ ٠٪ · ٧٠–٨٠٪ ⇒ ١٪ · ٨٠–٩٠٪ ⇒ ٢٪ · ٩٠–٩٥٪ ⇒ ٣٪ · ٩٥–١٠٠٪ ⇒ ٥٪.
معدل التحصيل لكل عميل = (إجمالي المُفوتر − المديونية القائمة) ÷ إجمالي المُفوتر (لقطة المديونية).

## قيود المصدر (شفافية — لا اختلاق)

الملفات المصدرية لا تحتوي على:
- **تكلفة** ⇒ لا هامش ربح إجمالي/EBITDA (تظهر البطاقة كـ«غير متاح»).
- **موازنة** ⇒ لا مقارنة فعلي مقابل موازنة.
- **تواريخ استحقاق على الفواتير** ⇒ **أعمار الديون تقديرية**: يُبقى «الجاري» كما في لقطة
  المديونية، وتُوزَّع «المتأخرات» على الشرائح حسب عمر آخر نشاط (صافي مدة السداد الافتراضية).
  مجاميع الشرائح تُطابق إجمالي المديونية بالضبط؛ فقط التوزيع داخل نطاق التأخر تقديري ومُعلَّم بوضوح.
- **إحداثيات جغرافية** ⇒ الخريطة الجغرافية مُستبدلة بتوزيع حسب المندوب/المنطقة.

## التحقّق

يطبع `build.py` تقريرًا يتحقق من: مطابقة الفواتير (١٠٠٪)، تطابق مجاميع البنود مع الإجماليات،
عدد الفواتير (٣١١) والعملاء (١١٦)، مجموع مبيعات العملاء = الإجمالي، مجموع مساهمة المنتجات ≈ ١٠٠٪،
احتساب ASP، تطابق شرائح الأعمار مع إجمالي المديونية، وحدود سلّم الحافز.
